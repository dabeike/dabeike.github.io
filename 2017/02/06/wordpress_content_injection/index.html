WordPress REST API 内容注入漏洞分析

Feb 6, 2017 | Hits

0x00 漏洞分析

wordpress支持REST API，自定义的route有很多功能，比如指定权限验证函数、指定主体函数（主体函数中不再验证用户操作权限）、指定参数等，如下图：

需要注意的是：

  • 如果未指定参数名，则默认为id（L80， exp: /wp-json/wp/v2/posts/1234 => id=1234）
  • 可以通过正常的$_GET、$_POST传参来覆盖伪静态的参数（exp: /wp-json/wp/v2/posts/1234?id=12345helloworld => id=12345helloworld）
  • 函数执行顺序为 permission_callback => callback（L95-96，即为先调用update_item_permission_check判断是否有权限，再调用update_item执行更新操作）

先看下update_item_permission_check函数，出现问题的点是L594。如果$post为false，不会抛出错误，而是继续走下面的流程，即绕过了权限验证。

看下get_post函数，怎么才能让$post参数为false呢？很简单，id=123ABC即可。

那么如果我们把id参数置为123ABC的时候，会不会对后续的修改产生影响呢？继续看update_item：

L623可以看到$request['id']被强制转换为int类型，不影响后续的执行。
